# Contributing to Kutumbly Sovereign OS

Thank you for your interest in Kutumbly. We follow a strict **Sovereign Development Protocol** to ensure the privacy and security of family data.

## 🔒 Security First
Kutumbly is a **Zero Cloud** product. We do not accept any contributions that:
- Introduce telemetry or tracking.
- Attempt to move data to external servers.
- Compromise the encryption layer (.kutumb files).

## 🛠️ Development Standards
- **Nomenclature**: Use standardized Sovereign Hub names:
  - Utsav (Events)
  - Sewak (Support Staff)
  - Cash (Finance)
  - Health (Wellness)
  - Saman (Inventory)
  - Vidya (Education)
- **Local First**: All logic must reside within the client (SQL-WASM).
- **Haptic Design**: UI must feel "physical" and premium, optimized for both mobile and desktop.

## 🚀 Getting Started
1. Fork the repository.
2. Follow the setup instructions in [README.md](README.md).
3. Create a branch for your feature: `git checkout -b feat/your-feature`.
4. Ensure all files include the official **Kutumbly Signature Block**.

## 📝 Commit Guidelines
Use clear, prefix-based commit messages:
- `FEAT`: New sovereign capabilities.
- `FIX`: Resolving bugs or build issues.
- `POLISH`: UI/UX enhancements or design refinements.
- `CHORE`: Repo maintenance or structural updates.
